data type

it is possible to specify the data type of each attribute directly, as in src-sql-example-1; alternatively, a domain can be declared, and the domain name can be used with the attribute specification. this makes it easier to change the data type for a domain that is used by numerous attributes in a schema, and improves schema readability.
for example, we can create a domain SSN_TYPE by the following statement:

CREATE DOMAIN SSN_TYPE AS CHAR(9);

we can use SSN_TYPE in place of CHAR(9) in src-sql-example-1 for the attributes Ssn and
Super_ssn of EMPLOYEE, Mgr_ssn of DEPARTMENT, and so on. a domain can also have an optional default specification via a DEFAULT clause. notice that domains may not be available in some implementations of SQL.
in SQL, there is also a CREATE TYPE command, which can be used to create user defined types or UDTs. these can then be used either as data types for attributes, or as the basis for creating tables.
