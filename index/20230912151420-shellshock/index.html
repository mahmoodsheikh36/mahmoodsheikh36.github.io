shellshock

a vulnerability (CVE-2014-6271) in bash versions 1.14 through 4.3 of the parent process can pass a function definition to a child shell via an environment variable the bash in the child process converts the value of the environment variable to a function so, it is supposed to parse the commands in the variable and not to execute them due to a bug in bash parsing logic, it executes some of the commands in the variable passing a bash function to a child process can be done with

export -f <function-name>

printing a function definition can be done with

declare -f <function-name>

exmaple usage:

sayHello() { echo hello; }
export -f sayHello
echo $$
bash -c '
echo $$
declare -f sayHello
sayHello
'

example exploit the shellshock bug starts in variables.c file of the bash source code in the initialize_shell_variables function which calls parse_and_execute
