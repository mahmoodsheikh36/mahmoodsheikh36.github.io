mimetypes

on linux desktops, the default program to open a file using xdg-open <file> is determined by its mimetype (or filetype)
notice that most programming languages and programs on linux also respect mimetypes and their handlers
to query the mimetype of a file:

xdg-mime query filetype /home/mahmooz
xdg-mime query filetype /etc/hosts
xdg-mime query filetype ~/test.py
xdg-mime query filetype ~/test.mp4

inode/directory
text/plain
text/x-script.python
video/mp4

to query the default application used for a mimetype:

xdg-mime query default inode/directory
xdg-mime query default text/plain
xdg-mime query default video/mp4

kitty-open.desktop
emacsclient.desktop
mpv.desktop

usually the configuration file of mimetypes and their corresponding applications are /usr/local/share/applications, /usr/share/applications and they are updated with the command update-desktop-database
but user overrides should be defined in ~/.config/mimeapps.list, the file format is simple and straightforward, heres (some of) mine as an example:

[Default Applications]
application/json=emacsclient.desktop;
application/octet-stream=emacsclient.desktop;
application/x-bittorrent=transmission-gtk.desktop;
image/jpeg=sxiv.desktop;
image/png=sxiv.desktop;
image/gif=sxiv.desktop;
inode/directory=Thunar.desktop;
inode/x-empty=emacsclient.desktop;
rust-toolchain.org=emacsclient.desktop;
snap=emacsclient.desktop;
src=emacsclient.desktop;
target=emacsclient.desktop;
text/html=google-chrome-stable.desktop;
text/plain=emacsclient.desktop;;
text/x-lisp=emacsclient.desktop;
text/x-shellscript=emacsclient.desktop;
video/mp4=mpv.desktop;
video/x-matroska=mpv.desktop;
x-scheme-handler/about=google-chrome-stable.desktop;
x-scheme-handler/http=google-chrome-stable.desktop;
x-scheme-handler/https=google-chrome-stable.desktop;
x-scheme-handler/unknown=google-chrome-stable.desktop;
application/pdf=org.pwmt.zathura.desktop;
inode/directory=emacsclient.desktop;

you dont need to run any commands after modifying your mimeapps.list
