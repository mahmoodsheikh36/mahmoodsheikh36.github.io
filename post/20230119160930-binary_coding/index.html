a binary coding is an injective function that maps each character of a finite set of characters to string of bits, given a character \(c\), and a binary coding \(k\), \(k( c)\) is called the coding of \(c\), and the number of bits in \(k( c)\) is denoted by \(|k( c)|\)

consider the following binary coding function for the set \(C=\{a,b,c,d,e,f\}\)

character \(c\)coding \(k( c)\)
a000
b001
c010
d011
e100
f101

the binary coding of a file \(F=c_1,c_2,\dots,c_l\) is a chain of codes of its characters, \(k(F)=k(c_1)\circ\dots\circ k(c_1)\)

\[ k_1(adf) = 000 \circ 011 \circ 101 = 000011101 \]

#+begin_definition the frequency of a character \(c \in C\) in a file \(F\), \(f(c,F)\) is the number of times that character \(c\) appears in \(F\)

let \(|k(F)|\) denote the number of characters in an arbitrary file \(F\) \[

k(F)=∑c ∈ Ck(c)f(c,F)

\]

suppose to we wanna encode a file \(F\) that contains 100 words over a 6-letter alphabet, we also assume that each character in \(F\) has a different frequency an efficient coding of \(F\) would match short binary strings with characters that have a high frequency, and long strings wirth letters that have lower frequency, the following table compares two codings of \(F\):

characterfrequency\(k_1\), fixed-length code word\(k_2\), variable-length code word

with this example table we can already see why variable length code is more efficient as \(k_2(F)=222,k_1(F)=300\)
