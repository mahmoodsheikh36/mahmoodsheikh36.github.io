dijkstra's algorithm

\(\DeclareMathOperator{\spn}{span}\) \(\DeclareMathOperator{\dom}{domain}\) \(\DeclareMathOperator{\ran}{range}\) \(\DeclareMathOperator{\rng}{range}\) \(\DeclareMathOperator{\img}{Im}\) \(\DeclareMathOperator{\adj}{adj}\) \(\newcommand\dif[1]{\:\textrm{d}#1}\) \(\DeclarePairedDelimiter\ceil{\lceil}{\rceil}\) \(\DeclarePairedDelimiter\floor{\lfloor}{\rfloor}\) \(\newcommand{\ihat}{\hat{\textbf{i}}}\) \(\newcommand{\jhat}{\hat{\textbf{j}}}\) \(\newcommand{\khat}{\hat{\textbf{k}}}\) \(\newcommand{\rhat}{\hat{\textbf{r}}}\) \(\newcommand{\thetahat}{\boldsymbol{\hat{\theta}}}\) \(\newcommand\brm[1]{\textbf{#1}}\) \(\DeclareMathOperator*{\argmax}{arg\,max}\ \(\DeclareMathOperator*{\argmin}{arg\,min}\)

an algorithm that finds the distances from a given vertex to all the other vertices in a weighted graph

relaxing step:
the method \(\textsc{Relax}\) is the building block of dijkstra’s algorithm, it checks whether the path we’ve taken to get to the vertex we’re currently at is shorter than a past path we’ve taken, and updates the distance array accordingly. in this note i refer to this as a relaxing step

\begin{alg} \KwIn{ \begin{enumerate} \item a graph $G=(V,E,w)$, the graph $G$ is weighted non-negatively ($w(e) \geq 0, e \in E$) \item a vertex $s \in V$ (the origin) \end{enumerate} } \KwOut{ for every vertex $v \in V$ two variables, $d[v],p[v]$ with the following restrictions \begin{enumerate} \item $d[v]=\delta(s,v)$ \item $p[v]$ is the second-to-last vertex in the minimal path from $s$ to $v$ \end{enumerate} } \SetKwFunction{dijkstra}{\textsc{Dijkstra}} \SetKwFunction{dijinit}{\textsc{Dij-Init}} \SetKwFunction{dijop}{\textsc{Dij-Op}} \SetKwFunction{rlx}{\textsc{Relax}} \fn{\dijkstra{G,s}} { \dijinit{G,s}\; \dijop{G,s}\; } \fn{\dijinit{G,s}} { $d[s] \gets 0$\; $p[s] \gets Null$\; \For{$v \in V - \{s\}$} { $d[v] \gets \infty$\; $p[v] \gets Null$\; } } \fn{\dijop{G,s}} { $PQ \gets V$ \tcp*[l] {priority queue} \While{$PQ \neq \varnothing$} { $u \gets PQ.extractMin()$ \tcp*[l] {vertex with minimal $d$ value} \For{$v \in adj[u]$} { \rlx{G,u,v}\; } } \Return $d,p$\; } \fn{\rlx{G,u,v}} { \If {$d[v] > d[u] + w(u,v)$} { $d[v] \gets d[u] + w(u,v)$\; $p[v] \gets u$\; } } \end{alg}

let \(G=(V,E,w)\) be a non-negatively weighted graph. assume that dijkstra is being applied to \(G\) starting at some vertex \(s\). at each step, for every vertex \(v\), if \(d[v] < \infty\) then in \(G\) there exists a path \(a\) from \(s\) to \(v\) and:

  1. \(|a|=d[v]\)
  2. if \(s \neq v\) then \(p[v]\) is the second-to-last vertex in the path \(a\)

let \(R=r_1,r_2,\dots\) be a series of relaxing steps that were applied during the execution of the algorithm. the proof is done by induction on \(i\) (index of relaxing steps)
base step (for \(i=0\)): before the application of \(r_1\), \(d[s]=0\), and for every other vertex \(v\), \(d[v]=\infty\) and so the claim holds true (in this case)
induction step: induction hypothesis: assume the claim is true after the execution of \(r_1,r_2,\dots,r_i\), and let \(r_{i+1} = \textsc{Relax}(u,v)\). for each vertex \(u \neq v\), \(d[u]\) doesnt change, and the claim follows trivially from the induction hypothesis. if \(r_{i+1}\) is an empty relaxing step, the claim is also true for \(v\). assume \(r_{i+1}\) isnt an empty step, in this case, after the relaxing step \(r_{i+1}\) is executed, \(d[v]=d[u]+w(u,v)\) and \(p[v]=u\), according to the induction hypothesis, there is a path from \(s\) to \(u\) whose length is \(d[u]\) and it follows that there is a path from \(s\) to \(v\) whose length is \(d[v]=d[u]+w(u,v)\), and the last edge in that path is \((u,v)\).

\(d[v] \geq \delta(s,v)\) is true for all \(v \in V\) at each step of the dijkstra algorithm, if during one step \(d[v] = \delta(s,v)\), the value of \(d[v]\) stays the same and doesnt change again during the execution of the algorithm

according to lemma 1, for each vertex \(v\) and at each step of the algorithm, if \(d[v] \neq \infty\), \(d[v]\) is the length of some path from \(s\) to \(v\). from the code it follows that the value of \(d[v]\) changes only by the relaxing method, and so this value can only get smaller, the correctness of the entailment follows from the definition of \(\delta(s,v)\) being the minimal path from \(s\) to \(v\).

let \(G\) be a non-negatively weighted graph, and let \(a=(s=v_0,v_1,\dots,v_l)\) be a minimal path in \(G\), then:

  1. \(\delta(s,v_i) \le \delta(s,v_{i+1})\) for each \(0 < i < l\)
  2. if executing a series of relaxing steps \(r_0,\dots,r_{l-1}\) along \(a\), such that \(r_i=\textsc{Relax}(v_i,v_{i+1}),0 \le i \le l-1\) then \(d[v_{i+1}]=\delta(s,v_{i+1})\)

note that the series of relaxing steps would be done in the written order, but between each two steps there might be more steps that get executed

proof for claim 1: we denote \(a_i=(s,v_1,\dots,v_i)\). \(w(v_i,v_{i+1})\geq 0\) for all \(i\) and it follows that \(|a_i|\le |a_{i+1}|\). every subpath of a minimal path is a minimal path and so \(|a_i|=\delta(s,v_i)\) for each \(i\)

proof for claim 2: left unwritten

main lemma for proof of correctness:
let \(G=(V,E,w)\) be a non-negatively weighted graph. upon applying dijkstra to \(G\), starting at \(s\), each vertex \(v \in V\), by the time it gets removed from \(PQ\), would have had its demands met.

the proof is by induction on the removal of vertices from \(PQ\)
base step: after the initialization \(d[s]=0\) and for each vertex \(v\), \(v \neq s\), its true that \(d[v]=\infty\). the first vertex that gets removed from \(PQ\) is \(s\). since the weights of all the edges arent negative, \(\delta(s,s)=0=d[s]\), and so \(s\) meets the first demand. the second demand of the vertex is empty as it should be (\(p[v]\)).
induction step: induction hypothesis: assume the demands are met for each of the vertex that were removed from \(PQ\) in the timestamps \(1,2,\dots,t\) and let \(v\) be the vertex that is removed from \(PQ\) at the timestamp \(t+1\). we have to prove that at the timestamp \(t+1\) the demands of the vertex \(v\) are met. given that every vertex in \(G\) is reachable from \(s\). let \(a\) be a minimal path from \(s\) to \(v\). assume that \(a=(v_0=s,v_1,v_2,\dots,v_l=v)\). let \(v_k,k<l\) be the last vertex in \(a\) that was removed from \(PQ\) before \(t+1\). according to the induction hypothesis, by the time of removal of \(v_k\) from \(PQ\), we know \(d[v_k]=\delta(s,v_k)\). right after, relaxing steps are applied from \(v_k\) to its neighbors, one of which is \(\textsc{Relax}(v_k,v_{k+1})\). the path \(a\) is minimal. according to lemma 2, after the execution of this relaxing step, we know \(d[v_{k+1}]=\delta(s,v_{k+1})\), consider the following cases:
case 1: \(v_{k+1}=v_l\), when \(v_k\) is removed from \(PQ\), the algorithm applies a relaxing step to each of \(v_k\)’s neighbors, one of which is \(v_{k+1}\), according to lemma 2, after the execution of \(\textsc{Relax}(v_k,v_{k+1})\), we know \(d[v_{k+1}]=\delta(s,v_{k+1})=\delta(s,v_l)\) and \(p[v_l]=v_k\) is the second-to-last vertex in the minimal path from \(s\) to \(v\).
case 2: \(v_{k+1}\neq v_l\), this happens when \(a=(s,v_1,\dots,v_k,v_{k+1},\dots,v_l)\) is a minimal path from \(s\) to \(v_l\), this might happen when the weights of the edges leading up to \(v_l\) are 0, which would give other vertices the same distance from the origin as \(v_l\), and so the algorithm could choose to relax some edges before others, and the “natural” order of traversal could change, but because we know \(\delta(s,v_l) \le d[v_l] \le d[v_{k+1}] \le \delta(s,v_l)\) it follows that \(d[v_l]=\delta(s,v_l)\). let \(u\) be the vertex that “discovers” \(v\). according to the code \(u\) exists in \(PQ\) before \(v\). according to the induction hypothesis the demands for \(u\) would already have been met, it follows that after the last relaxing step the demands for \(v\) would be met.

correctness of dijkstra:
assume dijkstra is applied to a non-negatively weighted graph \(G=(V,E,w)\) and an origin vertex \(s \in V\). the demands of every vertex \(v \in V\) are met. this follows from lemma 3

the set of edges
\[ T=\{(p[v],v) \mid v \in V - \{s\}\} \]
forms a distance tree

for each vertex \(v \in V\), in \(T\) there is a path from the origin \(s\) to \(v\). it follows that \(T\) is connected. in \(T\) there is exactly \(|V|-1\) and so \(T\) is a tree. we’ve already seen that for all \(v \in V\), the only path in \(T\) from \(s\) to \(v\) is a minimal path in \(G\), i.e. \(\delta_T(s,v) = \delta_G(s,v)\)

time complexity of dijkstra:
the cost of initialization (using \(\textsc{Dij-Init}\)) is \(\Theta(|V|)\)
the main loop (in \(\textsc{Dij-Op}\)) executes \(\Theta(|V|)\) times, on each iteration it chooses a minimal vertex and improves on all of its neighbors, if \(G\) isnt a directed graph, then during the relaxing step we test each edge twice, once in each direction. if \(G\) is a directed graph, each edge is tested once. so the cost of all relaxing steps together is \(\Theta(|E|)\). picking the minimal vertex using the straightforward method also takes \(\Theta(|V|)\) steps. the process of picking the minimal vertex is repeated \(|V|\) times so the total cost is \(\Theta(|V|^2+|E|)=\Theta(|V|^2)\)
for sparse graph, we can improve the time complexity by turning \(Q\) into a min heap, picking a minimal vertex would take \(O(\log|V|)\). the cost of all picks would be \(O(|V|\log|V|)\), on each of the \(|E|\) relaxing steps we would need to do a correction in the heap, the cost of all corrections would be \(O(|E|\log|V|)\), so the total cost would be \(O((|E|+|V|)\log|V|)=O(|E|\log|V|)\)

given a weighted graph \(G=(V,E,w)\) that represents a town’s junctions/roads
given \(L \subseteq V\) is a subset of junctions with streetlights
given \(s \in V\), propose an algorithm that given \(G,L,s\) would find the length of the shortest paths from \(s\) to each of the vertices in the graph, such that each path wouldnt contain more than 2 junctions with streetlights

\begin{alg} $V’ \gets V \times \{0,1,2\}$\; $E’ \gets \{((u,0),(v,0)),((u,1),(v,1)),((u,2),(v,2)) \mid (u,v) \in E, u \not \in L\} \cup \{((u,0),(v,1)),((u,1),(v,2)) \mid (u,v) \in E, u \in L\}$\; $w’((u,x),(v,y)) = w(u,v) \quad ((u,x),(v,y)) \in E’$\; \textsc{Dijkstra}(G’, (s,0))\; \Return $\min\{d((v,0)),d((v,1)),d((v,2))\}$ for each $v \in V$\; \end{alg}

complexity of this algorithm is \(\Theta(|V|^2)\)
