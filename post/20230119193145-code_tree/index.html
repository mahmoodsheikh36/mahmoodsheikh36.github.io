let \(k\) be a prefix code over the characters \(C=\{c_1,\dots,c_n\}\), the code tree of \(k\), \(T_k=(V_k,E_k)\) is a binary tree such that every inner node, \(v \in V_k\), one of the lines coming out of \(v\) are labelled by 0, and the other by 1. if from \(v\) comes out one line only, then its labelled 0 or 1.
each character in \(C\) is matched with a unique root-to-leaf path in \(T_k\), we denote the path that corresponds to \(c_i\) by \(a_i\). the leaf at the end of the path of \(a_i\) is tagged by \(c_i\) and the lines through \(a_i\), starting from the root to the leaves are tagged by the bits of the code \(k(c_i)\)
in other words: if \(k(c_i)=b_1,\dots,b_l\) then in \(a_i\) we have \(l\) lines, and the line of \(j\), \(1 \le j \le l\), is tagged by \(b_j\). the code trees of \(k_l\) and of \(k_2,T_{k_1},T_{k_2}\) are shown in the following diagrams

a tree code, \(T_k\), allows simple and efficient decoding of a file that has been encoded using \(k\), such that each character in the original file is decoded by processing \(T_k\) from the root to one of the leaves
for example, if we were to look in the trees we’d find that \(k_1(adf)=000\circ011\circ101=000011101\), to decode the first character in \(k_1(adf)\), we process the tree starting from the root \(T_{k_1}\), we progress using the lines that are tagged by 0,0,0, which are the 3 first bits in the encoded file, we arrive to the leaf that is tagged by \(a\), which is the first character in the encoded file
now the first 3 bits are removed from the encoded file and as such the decoding of the first character is done. continuing, the decoding of each of the characters is done by processing a root-to-leaf path, addition of the character that appears in the leaf to the decoded file, and removal of the bits from the original encoded file

1 cost of code tree

let \(F\) be a file over the set of characters \(C=\{c_1,c_2,\dots,c_n\}\), let \(k\) be prefix code of \(C\), with the code tree \(T_k\), for each \(c_i \in C\), we denote by \(f_i\) the frequency of \(c_i\) in \(F\), and call that the number of bits in \(k(c_i)\) is equal to the number of lines of \(a_i\). the cost of \(T_k\), in correspondance to the file \(F\), \(cost(T_k,F)\), is defined as the number of bits in \(k(F)\), so:
\[ cost(T_k,F) = \sum_{i=1}^{n} f_i \cdot |k(c_i)| = \sum_{i=1}^{n} f_i \cdot |a_i| \]
the code tree \(T_k\) is optimal in correspondance to \(F\), if there exists no prefix code \(k’\) such that \(cost(T_k,F) < cost(T_k,F)\)

2 representative leaf and leaf array

let \(F\) be a file over the set of characters \(C=\{c_1,c_2,\dots,c_n\}\), for each \(1 \le i \le n\), let \(f_i\) be the frequency of \(c_i\). the leaf that represents the character \(c_i\) is the node labelled by the ordered pair \(p_i=(c_i,f_i)\). the components of \(p_i\) are denoted by \(p_i.f\) and \(p_i.c\)
the leaf array of \(F\) is an array \(D[]\) of \(n\) elements, such that \(D[i]\) stores the leaf that represents \(c_i\), i.e. \((\forall 1 \le i \le n)[D[i]=p_i]\)

the leaf array contains all the necessary data to calculate the optimal code tree for \(F\)
