1 matrix multiplication using dynamic programming

refer to for the mathematics
given the matrices \(A_{i \times j} \times B_{j \times k} = C_{i \times k}\), the number of multiplication operations needed to compute \(C\) is \(i \cdot j \cdot k\), the number of addition operations needed to compute \(C\) is \(i \cdot (j-1) \cdot k\) which is close but a little smaller

given three multipliable matrices \({M_1}_{x_1 \times y_1},{M_2}_{x_2 \times y_2},{M_3}_{x_3 \times y_3}\), we want to find the number of multiplication operations required to compute \(M_{13}=M_1 \times M_2 \times M_3\)
there are a few “timings” (order of application) in which we can apply the multiplication:

  1. the timing (1,2) which denotes the order of multiplication \((M_1 \times M_2) \times M_3\), which takes \(x_1x_2y_2+x_1x_3y_3=x_1x_3(x_2+y_3)\) operations
  2. the timing (2,1) which denotes the order \(M_1 \times (M_2 \times M_3)\) which takes \(x_2x_3y_3+x_1x_2y_3=x_2y_3(x_3+x_1)\) operations

we denote \(M_{ij} = M_i \times \dots \times M_j\) for \(1 \le i < j \le n\)
the number of the operation between \(M_i\) and \(M_{i+1}\) would be \(i\)
every permutation of the sequence \(S_{n-1}=1,2,\dots,n-1\) corresponds to a particular timing each with a different cost but the same resulting matrix
given the multiplication of matrices \(M_{1n} = M_1 \times M_2 \times \dots \times M_n\), we define the cost of timing \(s\), \(Cost(s,M_{1n})\) as the number of multiplication operations needed to compute \(M_{1n}\) according to \(s\)
the cost of finding \(M_{1n}\) is defined as the minimal cost of a given timing
\[ Cost(M_{1n}) = \min_{s \in S_{n-1}}(Cost(s,M_{1n})) \]
we want an algorithm that finds the best timing (one with the minimal cost). a straightforward algorithm goes through each timing and finds the one with the minimal cost which would take \((n-1)!\) time because a timing is a of a sequence of \(n-1\) numbers
we try a recursive solution. for each \((i,j),i \le j\), we denote by \(mc(i,j)\) the number of minimal multiplications required to find \(M_{ij}\). with this notation, the cost of computing \(M_{1n}\) is \(mc(1,n)\)
the algorithm is based on the following 2 lemmas:

\[ (\forall 1 \le i \le j \le n)[mc(i,i)=0 \text{ and } mc(i,i+1)=x_ix_{i+1}y_{i+1}] \]

\[ (\forall 1 \le i \le j \le n)[\text{if } j>i+1 \text{ then } mc(i,j)=\min_{k=1}^{j-1}(mc(i,k)+mc(k+1,j)+x_ix_{k+1}y_j) \]


if we were to follow the steps of the algorithm, we’d notice the following:
to find mc(1,5) the algorithm has to find the minimum of:

\begin{gather*} mc(1,1) + mc(2,5) + x_1x_2y_5\\ mc(1,2) + mc(3,5) + x_1x_3y_5\\ mc(1,3) + mc(4,5) + x_1x_4y_5\\ mc(1,4) + mc(5,5) + x_1x_5y_5 \end{gather*}

and to find mc(2,5) we’d need to find the minimum of:

\begin{gather*} mc(2,2) + mc(3,5) + x_2x_3y_5\\ mc(2,3) + mc(4,5) + x_2x_4y_5\\ mc(2,4) + mc(5,5) + x_2x_5y_5 \end{gather*}

we’d notice for example that mc(3,5) and mc(4,5) are being calculated twice, so the algorithm does some repeated work that can be reduced using dynamic programming
the algorithm has a lower bound of \(2^{n-1}\) and while it is better than the straightforward solution, it is still inefficient
each of the two arguments that the function mc takes can vary between n values, so the total number of permutations of arguments it can take is \(n^2\), so we use a lookup table of size \(n^2\) and use it to store the results of executing the function mc with each possible configuration of the arguments
